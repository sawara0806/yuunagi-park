# ゆうなぎ公園 — プロジェクト手引書（引き継ぎ用）

このファイルは次の担当（人間・AIどちらでも）への引き継ぎ書。
**コードを触る前に必ず全部読むこと。** 特に「不変条件と落とし穴」。

## 1. プロジェクトの目的

SNSなど情報過多に疲れたインターネットユーザーに、**実際に公園で過ごしたような**
安心・リラックス体験を届けるWebサイト。通知・スコア・タイマー・実績は永遠に入れない。

- 題材: ユーザー提供の実在の街区公園の写真（レンガ円形広場・ベンチ列・大木と
  ヤブランの植え込み・金網フェンス・外階段付きアパート・遠くにビル）
- 表現: **リアルで美しいドット絵**を最優先。動きは質を損なわない範囲で足す
- 画風の参照: ピクセルアーティスト「せたも」
  （静かな風景と光・限定パレット・影は暗くせず色相を青紫へシフト・
  「少し風が吹くだけで絵に入り込める」＝最小限の動きで没入・背景に小さな物語）

## 2. 構成（ビルドなし・依存ゼロの素のJS）

```
pixel-park/
  index.html        入園ゲート・canvas(480×270)・歩行ボタン
  style.css         pixelated拡大・UI・overscroll対策（消すな）
  js/layout3d.js    公園と周辺の配置の唯一の設計図（定数 LAYOUT。propsとcritters は型つきリスト）
  js/assets3d.js    全ドット絵素材の手続き生成。LAYOUTを参照して地面の焼き込み影を作る。
                    天気で変わる値（グレーディング/空/雲/霞/濡れ/粒子/音）は冒頭の
                    WEATHER レジストリに集約（curWeather()で参照。ENV.weatherを直接分岐しない）
  js/render3d.js    ソフトウェアレンダラ R3
  js/audio3d.js     環境音（Web Audio合成の meowInto と AUDIO3）。park3d より先に読み込む
  js/park3d.js      シーン組立・移動・入力・メインループ。LAYOUTを参照してシーンを組む
  v1/               旧・横スクロール2D版（せたも風・4時間帯・豊かな音風景）
```

別作品 `~/Documents/virtual-park/`（こもれび公園・canvas1枚）もある。両方残す方針。

### レンダラ（render3d.js の R3）
- **地面**: mode7式フロアキャスティング。Uint32テクスチャ直読み＋行ごとの距離霞み
- **壁**: 透視補正テクスチャ壁（列ごとに1pxずつ縦引き）。建物・フェンス・生け垣・電線
- **箱**: 外向きの面＋天面（走査線塗り）。ベンチ・掲示板。どこから見ても矛盾しない
- **ビルボード**: 回転対称な物だけ（木・茂み・鳩・電柱・公園灯）
- **見上げ/見下ろし**: y-shear方式。`cam.hor`（地平線のy、26..234）を上下するだけ
- 描画順: 空 → 地面 → 全アイテムを距離ソートして遠→近（painter's algorithm）
- **できないこと（企画時に注意）**: 動的光源・実行時のテクスチャ変化（色や光は
  すべて生成時に焼き込み。変えるには rebuildWorld 約0.7秒）、半透明ブレンド、
  傾いた面・屋根形状（箱は垂直壁＋水平天面のみ）、回転非対称なビルボード
  （正面しか無い物は横から見ると破綻する。必要なら箱か壁で作る）

### 素材（assets3d.js）
- 全て起動時にコードで生成。画像ファイルは1枚も無い
- 配置定数 `L`（木・ベンチ・花壇・フェンス等の座標）を
  **地面テクスチャの焼き込み影とシーン組立の両方が参照**する
- 地面: 40m×40m・24px/m。木漏れ日・ベンチ・生け垣の影は焼き込み（太陽は北西、影は南東へ）
- 空: 幅2325px（=2π×投影距離370）の360°ストリップ。yawで横スクロール。
  山なみは `sin(x/w*2π*整数)` の周期関数なので左右がwrapで必ず繋がる

## 3. 不変条件と落とし穴（全部実際に踏んだもの）

1. **cam.hor は必ず整数で描画に使う。** ドラッグで小数になる。小数のまま
   フロアキャスティングに入ると書き込み行が壊れ「地面がバグる」。
   現在は `R3.render()` 冒頭で `Math.round`。この防御を外すな
2. **歩行の揺れは bobAmp（振幅）を減衰させる。** bobT（位相）を減衰させると
   立ち止まった後もしばらく揺れ続ける
3. **html/body の `overscroll-behavior:none` と `touch-action:none` を消すな。**
   ドラッグ見回しがブラウザの「戻るスワイプ／引っ張って更新」に化けて
   ページ遷移してしまう（「出口を出るとトップページに移動」の正体）
4. **配置はすべて `js/layout3d.js` の `LAYOUT` が単一の真実。** 影・コライダ・シーン・
   座席アンカーはそこから自動導出されるので、再設計は layout3d.js の編集だけで完結する
   （旧ルール「Lと影とコライダの3箇所を手で揃える」は廃止。`L` は assets3d.js に残る
   互換エイリアスなので新規参照は使わないこと）。新しい種類の物を足すときは
   park3d.js の型別レジストリ（buildScene内の建物ループ・structures分岐・
   props の PROP_SPRITE レジストリ）に builder を1つ足す。
   **地面焼き込み影で「手で合わせている」箇所の一覧**（座標を動かすとき注意）:
   - 木の影: 位置は自動追従だが、影の中心は木から**南東へ (+1.9, +1.4) の固定オフセット**
     （assets3d.js buildFloorTex内）。太陽方位を変えたければここ
   - ベンチの影: bench.x から **x−0.65〜+1.15 / benchZ+0.3〜+1.05 の固定矩形**を焼く
   - 生け垣・トイレの影: `LAYOUT.hedges[].shadow` / `LAYOUT.structures.toilet.shadow`
     に実測の矩形をそのまま持たせてある（壁ジオメトリからの自動計算ではない。
     本体を動かしたら shadow の矩形も手で合わせる）
   - 夜の公園灯の光だまり: 中心は `LAYOUT.props` の lamp から自動導出（追従する）
5. ベンチの向き: 背もたれは北（生け垣側、-z）、座面は南（広場側、+z）。
   一度逆に作って気づかなかった
6. **壁は必ず `pushWall()` 経由で登録する（直接 WALLS.push しない）。**
   奥行きソートは壁の中点1点なので、長い壁のままだと近くの木やベンチと
   前後が逆転する（実際に起きた）。pushWall が2.5m単位に分割し、
   `u0`/`texLen` でテクスチャの継ぎ目も守る
7. 木の葉群はスプライト上端で切れると真っ平らになる。
   生成時に `py = max(cr*0.8+3, …)` でクランプ済み
8. FOVを変えるなら空の幅も変える（空幅 = 2π×PD）
9. **地平線の行（y == hor）はフロア側で霞色に塗る。** 空は hor-1 まで、
   フロアの距離計算は hor+1 からなので、塗り忘れると前フレームの残骸が
   1pxの黒い線になって地平線に残る（実際に起きた。renderFloor冒頭で対処済み）
10. 見回し: マウスはポインターロック（クリックでON/OFFトグル・Escでも解除・標準FPS向き）、
   タッチはドラッグ（画面を掴む向き＝FPSと逆）。両方の経路があるので
   感度や向きを変えるときは park3d.js の2箇所を揃えて直すこと
11. 枝や細い構造物は**実線**で描く。点線（ステップ間隔を空けた描画）だと
   近距離で拡大されたとき骨だけの扇に見える（ケヤキで実際に起きた）
12. 南の門を出る（z > 10.3）と `leavePark()` でタイトルへ戻る仕様。
   園外は歩けない。INSIDE フラグが移動を止め、再入園でリセットされる

## 4. 開発・検証の手順（macOSこの環境固有）

1. **配信**: `~/Documents` はTCCでプレビューサーバーから読めない（404になる）。
   プロジェクト一式をセッションのスクラッチパッドへコピーし、
   `Documents/.claude/launch.json` の `pixel-park` 設定（python3 http.server、port 8642、
   `--directory` をそのスクラッチパッドに向ける）で起動する。
   **スクラッチパッドのパスはセッションごとに変わる**ので launch.json を都度更新
2. **編集はDocuments側**で行い、変更のたびに `cp` で同期
3. **ブラウザはJSを強くキャッシュする**。反映されない時は
   `for f of [...] await fetch(f,{cache:"reload"}); location.reload()` を eval で実行
4. **プレビューはバックグラウンドでrAFが止まる**。時間待ちは無意味。
   `window.PARK` デバッグフックで決定的に進める:
   - `PARK.step(n)` … nフレーム進める
   - `PARK.cam.x/z/yaw/hor` … テレポートと視点
   - `PARK.input.fwd = true; PARK.step(60)` … 歩行テスト
   - 性能: `t0=performance.now(); PARK.step(120); (performance.now()-t0)/120`
     （現状 約1.4ms/frame。3ms超えたら退行）
5. 音はヘッドレスで聴けない。`PARK.audio.ctx.state === "running"` の確認まで。
   実聴はユーザーに依頼する
6. **リリース（push）のたびに index.html の `?v=` を必ず上げる**（例: 20260711a → b）。
   GitHub Pages は10分キャッシュのため、これを忘れると閲覧者側で
   「古いHTML＋新しいJS」の食い違いが起き、起動エラーで入園ボタンが死ぬ事故になる
   （2026-07-11に実際に起きた。バージョン付きURLならHTMLとJS/CSSが常に揃った組で読まれる）

## 5. デザイン言語（画の一貫性を守るルール）

- 内部解像度480×270固定。整数座標に描く。拡大はCSS `image-rendering: pixelated`
- 影は「暗くする」でなく青へ振る: `r*0.62, g*0.66, b*0.78`
- 遠景は霞ませる: 地面は距離霞み自動。壁テクスチャは生成時に
  `rgba(212,222,230, 0.08〜0.16)` を上掛け
- 大面積には必ずノイズ（±5〜10）とディザ（`(x+y)&1`）。無地の面を作らない
- 経年の気配を1割混ぜる: タイルの汚れ・レンガの欠けと色あせ・目地の草・苔・雑草
- 新しい建物テクスチャには接地影（下端5pxを暗く）を忘れない

## 5.5 既知の問題点（2026-07-10 監査。qa-reviewer による file:line 付き所見）

- **OGP/メタ情報が皆無**（index.html:3-11）。description・og:*・favicon・theme-color なし。
  URLを共有するとカードが白紙になる。og:image用の画像アセットも0枚（要スクショ作成）
- README / LICENSE なし。せたも参照など画風の出典も未記載

- `#hint` 要素が未使用のまま残存（index.html / style.css にあるがJSから参照ゼロ）
  → **P5の着席プロンプト表示先として流用する**
- ネコの近接「ニャー」が夜ゲートを無視（park3d.js の接近ループに `ENV.mode !== "night"` がない。
  「夜は鳥が鳴かない」設計と不整合）
- keydown の preventDefault が入園前のタイトル画面でも発火（INSIDE を見ていない）
- `requestPointerLock()` はPromiseを返すブラウザがあり、同期try/catchでは失敗を拾えない

- 縦持ちスマホでcanvasが小さな帯になる（横画面誘導なし）→ Step 4
- トグルボタンに aria-pressed なし・音ノードの長時間リークは未計測（ブラウザで要確認）

### ~~Step 3 — 雨~~（実装済み 2026-07-11、たまに降る化 2026-07-12）
実装（今後の調整の手がかり）:
- `ENV.weather = "clear" | "rain"`。**入園ボタンを押した瞬間に25%の確率で雨**
  （切替UIは無い。デバッグ用に `PARK.setWeather("rain"|"clear")` が残してある）

- 雨音は常設ノイズ1系統（this.rainG、ゲインで開閉）。雨天は風1.2倍・雀0.3倍・
  ハト/ヒヨドリ/セミは鳴かない
- ※「訪問者の記憶」案は滞在記録がスコア化に滑り§7に触れるリスクがあるため見送り

### ~~P3: 音風景の拡充~~（昼ぶんは実装済み 2026-07-08）
実装内容（すべてWeb Audio合成・AUDIO3.out()で距離ゲイン＋相対角パン）:
- **風・葉ずれがP2の風 `windNow(t)` と連動**（木に近いほど葉ずれが近い）
- 雀のさえずり／**鳩の「クルックー」**（園内の鳩の位置から）／
  **ヒヨドリ「ヒーヨヒーヨ」**（木から）／**ミンミンゼミ**（夏の昼・まれに一節）／
  **カラス**（遠くから）／**車が南の道を通過**（パンが左右へ流れる・道路に近いほど大きい）
- スケジューラは AUDIO3.update() 内のタイマー群（birdT/cooT/bulbulT/semiT/crowT/carT）
- 「遠くの子どもの声」は帯域ノイズでは雑音に聞こえるリスクが高く**見送り**。
  やるなら要実聴で慎重に
- 夕・夜ぶん（ヒグラシ・チャイム・コオロギ）は **P4の時間帯実装とセット**で。
  v1/js/audio.js に実証済みシンセがある
- 注意: initで作る常設ノードは必ず `this.xxx` に持たせること
  （ローカル変数のままだと update から参照できず落ちる。leafGで実際に起きた）

### ~~P4: 時間帯 + 季節~~（実装済み 2026-07-08）
仕組み（今後の調整の手がかり）:
- `ENV = { season, mode }`（assets3d.js冒頭）。**入園ボタンを押した瞬間の実際の
  月と時刻で決まる**（envFromDate。タイトル放置後や再入園でもその時点の値になる）。
  切替UIボタンは2026-07-10に撤去（ユーザー判断: 実時間に即すのが本義）。
  デバッグ用に `PARK.setEnv(season, mode)` と `PARK.envFromDate(date)` は残してある。
  変更時は `rebuildWorld()` が全素材とシーンを再生成（実測 約0.7秒）
- **時間帯 = グレーディング**: `sheetA.px/rect` が `gradeColor()`（MODE_GRADE）を通す。
  発光（灯り・点いた窓）は `pxRaw/rectRaw` でバイパス。
  空だけは `GRADE_BYPASS` でモード専用の色を直接描く（朝焼け・夕日・星と三日月）
- **季節 = 生成内容**: 木の葉パレット `LEAF_PAL`（クスノキ=常緑でトーンだけ変化、
  ケヤキ=春若葉/夏緑/秋紅葉/冬は裸の梢）。春は treeSm3 がサクラに差し替わり
  SAKURA_POS から花びらが舞う。秋は落ち葉が木の下に濃く積もる（地面に焼き込み）。
  冬は雪（LEAVES流用・ゆっくり直下）。サツキの花は春夏のみ
- 夜: 公園灯の光だまりを地面に焼き込み＋灯具は発光色、建物と遠景ビルの窓に灯り
- 音のゲート: セミ=夏昼、ヒグラシ=夏夕、コオロギ=夜（冬以外）、
  夕方切替時にチャイム、夜は鳥が鳴かない。ネコは近づくと「ニャー」（クールダウン18秒）
